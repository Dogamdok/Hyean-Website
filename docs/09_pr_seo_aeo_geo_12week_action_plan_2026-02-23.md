# HYEAN 12주 실행 계획 (PR + SEO + AEO + GEO)

> Date: 2026-02-23  
> Scope: `apps/hyean-web` + 외부 PR 채널 + 검색/AI 노출 운영

## 1) 목적

1. 웹사이트를 “문의 전환형 신뢰 자산”으로 고도화
2. 검색엔진(SEO)과 생성형 검색(AEO/GEO)에서 혜안 엔터티 신뢰도 강화
3. 사례 기반 PR로 브랜드 검색 수요를 선순환 구조로 전환

## 2) 실행 원칙

1. 주장보다 근거: 수치/출처/업데이트일을 모든 핵심 페이지에 유지
2. 단일 문장 인용성: AI 응답에 그대로 인용 가능한 사실형 문장 배치
3. 채널 분업: 사이트(근거 허브) + PR(수요 창출) + 검색(유입) + 문의(전환)

## 3) 12주 로드맵

## W1-W2 (기술/신뢰 기반)
1. robots/sitemap/핵심 메타/구조화데이터 배포 상태 고정
2. 대표 사례 6건 근거 블록(지표·출처·업데이트일) 완성
3. 문의 퍼널 이벤트(`view_contact`, `start_inquiry`, `submit_inquiry`) 계측

## W3-W4 (인덱싱/콘텐츠 정렬)
1. Search Console/Bing 등록, 대표 URL 우선 색인 요청
2. 인사이트 4건 주장-근거 연결 문장 재정비
3. 내부링크 재배치(홈/서비스/사례/문의 동선)

## W5-W8 (AEO/GEO 확장)
1. FAQ/질문형 블록 확대(문제 정의형, 의사결정형 질의)
2. “문제-개입-성과” 3문장 템플릿을 사례 페이지 공통 적용
3. AI 인용 모니터링 샘플 질의군 운영(브랜드/주제/지역 조합)

## W9-W12 (PR 연계 확장)
1. 월 1회 사례형 보도자료 배포(문제-개입-성과 구조)
2. 분기 1회 인터뷰/기고(실행 프레임 중심)
3. 유입-문의-미팅 전환 리포트로 다음 분기 백로그 확정

## 4) 채널별 핵심 Task

## Website/SEO
1. 페이지별 canonical/OG/twitter/JSON-LD 유지
2. `scripts/seo_aeo_geo_healthcheck.sh` 주 1회 실행
3. 색인 대상 큐(`scripts/generate_indexing_targets.sh`) 주 1회 갱신

## AEO/GEO
1. 사례별 핵심 수치 3개와 증빙 출처 연결
2. 용어 정의 블록 운영(다기관 협업, 실행 설계, 성과관리)
3. 질문형 답변 블록(FAQ) 월 2회 업데이트

## PR/홍보
1. 사례 릴리즈 1건/월
2. 성과 수치가 포함된 카드형 요약 자산 2건/월
3. 외부 노출 콘텐츠에 항상 원문 출처 URL 첨부

## 5) KPI 운영

## 획득(Acquisition)
1. 비브랜드 검색 노출/클릭/CTR
2. 대표 사례 상세 유입 세션

## 신뢰(Trust)
1. 사례 페이지 체류시간
2. 출처 링크 클릭률
3. FAQ 상호작용률

## 전환(Conversion)
1. 문의 시작률
2. 문의 완료율
3. 문의→미팅 전환율

## AI 검색(AEO/GEO)
1. 브랜드+주제 질의의 AI 응답 내 혜안 언급률
2. 핵심 수치/근거 문장 인용 정확도

## 6) 이번 스프린트 완료/잔여

완료:
1. 기술 SEO 핵심(robots/sitemap/구조화데이터) 반영
2. 대표 사례 6건 근거 블록 강화
3. 문의 퍼널 이벤트 계측 반영
4. 복구 이미지 캐시 우회 적용(청년마을 4·5번째 파일명 버전 분리)

잔여:
1. Search Console/Bing 실등록 및 1차 색인 요청
2. 월간 리포트 자동화(콘솔 데이터 + 문의 퍼널 결합)
